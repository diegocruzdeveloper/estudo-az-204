Resumo de Conceitos para a Prova de Certificação Azure (AZ-204)

✅ Desenvolver soluções de computação do Azure (25-30%)

Contêineres e Aplicativos de Contêiner

  • Azure Container Registry (ACR): Crie repositórios para armazenar imagens de container.
  • Publicar imagem no ACR: `docker push <registry>/<imagem>` após login e tag.
  • Azure Container Instances (ACI): Executa contêineres sem orquestrador; rápido e simples.
  • Azure Container Apps: Execução gerenciada de microsserviços baseados em contêiner com autoscaling.
Ver perguntas sobre Contêineres ➔

Azure App Service

  • App Service Plan: Define recursos (CPU/RAM).
  • Web Apps / API Apps: Hospedagem de aplicações web e APIs.
  • Diagnóstico e logs: Application Insights, Log Stream, Web Server Logs.
  • Deploy: GitHub Actions, Azure DevOps, FTP, CLI.
  • Configurações: TLS, conexões de string, variáveis de ambiente.
  • Auto Scale: Por CPU, memória ou filas.
  • Deployment Slots: Slots de produção/staging com swap.
Ver perguntas sobre App Service ➔

Azure Functions

  • Triggers: Timer, HTTP, Blob, Queue, Event Grid.
  • Bindings: Entrada/saída (ex: ler blob, gravar no Cosmos).
  • Durable Functions: Para workflows com estado.
Ver perguntas sobre Azure Functions ➔

✅ Desenvolver para Armazenamento do Azure (15-20%)

Azure Cosmos DB

  • APIs: SQL, MongoDB, Cassandra, Table, Gremlin.
  • Operar via SDK: CRUD com .NET, Java, Python etc.
  • Níveis de Consistência: Strong, Bounded Staleness, Session, Consistent Prefix, Eventual.
  • Feed de alteração (Change Feed): Detecta mudanças para processar eventos em tempo real.
Ver perguntas sobre Cosmos DB ➔

Azure Blob Storage

  • Propriedades/Metadados: Usados para armazenar dados extras.
  • SDK: Permite CRUD de blobs.
  • Políticas de ciclo de vida: Automatiza arquivamento/deleção.
  • Segurança: SAS Token, RBAC, Firewalls.
Ver perguntas sobre Blob Storage ➔

✅ Implementar a segurança do Azure (15-20%)

Autenticação e Autorização

  • Microsoft Identity / Entra ID (Azure AD): OAuth2, OpenID Connect, MSAL.
  • App registrations: Para apps acessarem APIs com tokens.
  • SAS (Shared Access Signature): Tokens com permissões limitadas.
  • Microsoft Graph API: Acesso a dados do M365 e Azure AD.
Ver perguntas sobre Autenticação ➔

Azure Key Vault / App Configuration

  • Azure Key Vault: Armazena secrets, keys, certificados com controle de acesso.
  • App Configuration: Gerencia configurações centralizadas para apps.
  • Identidades Gerenciadas: Azure fornece identidade gerenciada para recursos acessarem secrets sem armazenar credenciais no código.
Ver perguntas sobre Key Vault e App Config ➔

✅ Monitorar e solucionar problemas (10-15%)

Application Insights

  • Telemetria: Desempenho, exceções e uso do aplicativo.
  • Métricas / Logs / Rastreamentos: Insights de uso e problemas.
  • Alertas e Testes Web: Simulação de chamadas e notificação em caso de falhas.
Ver perguntas sobre Monitoramento ➔

✅ Conectar e consumir serviços (20-25%)

API Management (APIM)

  • Instância de APIM: Gerencia e protege APIs.
  • Documentação e acesso: Developer Portal.
  • Políticas: Cache, rate limit, transformação de URL/cabeçalhos.
Ver perguntas sobre API Management ➔

Event-driven Solutions

  • Event Grid: Eventos baseados em push (como blob criado).
  • Event Hub: Ingestão de dados em massa (telemetria, IoT).
Ver perguntas sobre Soluções por Eventos ➔

Message-based Solutions

  • Service Bus: Mensageria com suporte a ordenação, dead-letter e FIFO.
  • Queue Storage: Fila simples, baseada em armazenamento.
Ver perguntas sobre Soluções por Mensagens ➔
